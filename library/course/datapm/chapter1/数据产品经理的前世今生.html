# 3. 数据产品经理核心价值

  • 推送数据驱动型企业
  • 数据资产管理与沉淀
  • 数据能力认知和抽象

# 企业数据化进程
  • 高度认知：用户对数据分析体系及查询技能基本掌握，用户可基于数据工具构建个性化数据
  • 索取：用户获取个性化的数据需求，企业具备成型的数据体系：数据链路、数仓模型
  • 填鸭：通过数据团队输出数据结果，教育用户使用数据并引导数据可视化（报表数据可视化）
  • 萌芽：对数据没有概念，只知道指标/kpi
# 企业数据框架

  • 用户数据中心 - 人
    • 用户分析：活动、业务、产品：产品模式、业务创新
    • 个性化推荐：数据筛选 - 精准：降低成本
    • 营销投放：数据筛选 - 精准投放，降低成本，避免信息爆炸
  • 业务数据中心
    • 面向业务：订单、商品、财务、客服、物流
    • 业务画像：B端画像、商品产品画像、流程体系画像

业务分析与驱动：提高业务粘性、提高业务增长

# 数据体系建设
  • 数据分析体系：业务指标体系、用户标签体系
  • 数据产品体系：数据应用、数据管理、埋点采集、探索分析
  • 数据链路及架构：数据采集、数据集成、数据仓库、数据质量、存储计算、算法模型

业务数据：引导业务功能方向

数据产品化：降低数据使用成本，异域推广，bi报表、adhoc即席查询
